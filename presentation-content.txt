SLIDE 3: SOLUTION OVERVIEW
--------------------------------------------------------------------------------
Title: Solution Overview

Phishing Finder is an intelligent browser extension that:
• Automatically scans every website you visit
• Uses AI (Google Gemini) to detect phishing sites
• Blocks malicious websites in real-time
• Warns users about risky websites
• Maintains a shared database for faster detection
• Provides admin dashboard for monitoring

Key Benefits:
✓ Real-time protection
✓ AI-powered analysis
✓ Zero user intervention required
✓ Shared knowledge across all users
✓ Comprehensive audit logging

SLIDE 4: SYSTEM ARCHITECTURE
--------------------------------------------------------------------------------
Title: System Architecture

Three-Tier Architecture:

1. FRONTEND (Chrome Extension)
   - Browser extension for real-time protection
   - User interface for authentication
   - Warning banners and blocking pages

2. BACKEND (Next.js API)
   - RESTful API endpoints
   - AI-powered URL analysis
   - Database caching layer
   - Authentication service

3. DATABASE (PostgreSQL)
   - URL check results cache
   - User management
   - Audit records

4. AI SERVICE (Google Gemini)
   - LangChain framework integration
   - Website content analysis
   - Phishing detection logic

Flow: User → Extension → API → Database → AI Service

SLIDE 7: BACKEND API - URL CHECKING
--------------------------------------------------------------------------------
Title: Backend API - URL Checking Service

ENDPOINT: POST /api/check-url

Features:
• Base domain extraction (normalizes URLs)
• Database cache lookup (fast response)
• AI-powered analysis for new URLs
• Result caching for future requests
• Confidence scoring
• Detailed reasoning provided

Process Flow:
1. Extract base domain from URL
2. Check database cache
3. If cached → return immediately
4. If new → call AI service
5. Store result in database
6. Return analysis result

Performance:
• Cached URLs: < 50ms response time
• New URLs: 2-5 seconds (AI analysis)

SLIDE 18: DATABASE SCHEMA
--------------------------------------------------------------------------------
Title: Database Schema Design

THREE MAIN TABLES:

1. USERS TABLE
   • id (UUID, Primary Key)
   • email (Unique)
   • passwordHash
   • role (USER/ADMIN)
   • timestamps

2. URL_CHECKS TABLE
   • id (UUID, Primary Key)
   • url (Unique, Indexed)
   • isPhishing (Boolean)
   • hasWarning (Boolean)
   • warningType, warningSeverity, warningReason
   • confidence (Float)
   • reason (Text)
   • userId (Foreign Key)
   • timestamps

3. AUDIT_RECORDS TABLE
   • id (UUID, Primary Key)
   • userId (Foreign Key → Users)
   • url
   • isPhishing, hasWarning
   • warningType, warningSeverity, warningReason
   • action (BLOCKED/WARNING_SHOWN/ALLOWED)
   • ipAddress, userAgent
   • visitedAt, timestamps

SLIDE 21: USER FLOW - URL SCANNING
--------------------------------------------------------------------------------
Title: User Flow - URL Scanning Process

STEP 1: USER NAVIGATES TO URL
• User types URL or clicks link
• Extension intercepts request

STEP 2: CHECK CACHE
• Extension extracts base domain
• Checks local cache first
• If cached → instant result

STEP 3: API CALL (if not cached)
• Extension calls backend API
• Shows scanning page
• Backend checks database cache

STEP 4: AI ANALYSIS (if needed)
• Backend calls LangChain service
• AI analyzes URL
• May fetch website content
• Returns phishing/warning result

STEP 5: RESULT HANDLING
• If phishing → Block page shown
• If warning → Warning banner displayed
• If safe → Normal navigation
• Result cached for future

Screenshot: <url-scanning-flow>
